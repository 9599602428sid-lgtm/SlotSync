# Contributing to SlotSync

Thank you for your interest in contributing! 🎉

## Quick Start

1. **Fork** this repository
2. **Clone** your fork: `git clone https://github.com/YOUR-USERNAME/slotsync.git`
3. **Open** `index.html` in your browser — no build step needed
4. **Make changes** and test in browser
5. **Submit** a pull request

## Code Style

- The entire app lives in `index.html` (HTML + CSS + JS in one file)
- CSS variables are defined in `:root` — use them, don't hardcode colors
- JavaScript uses vanilla ES6+ — no frameworks or libraries
- Keep functions small and clearly named
- Comment non-obvious logic (especially in the slot-matching algorithm)

## What to Work On

Check the [Issues](../../issues) tab for open bugs and feature requests.
The [Roadmap](README.md#-roadmap) in the README lists planned features.

## Pull Request Checklist

- [ ] Tested in Chrome and Firefox
- [ ] Tested on mobile viewport (375px width)
- [ ] No console errors
- [ ] README updated if behaviour changed

## Questions?

Open a [GitHub Issue](../../issues/new) and tag it with `question`.
